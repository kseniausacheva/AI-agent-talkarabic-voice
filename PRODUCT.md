# Product

## Register

product

## Users

Менеджеры по продажам, project-менеджеры и бизнес-аналитики, которые ведут созвоны с клиентами и должны заполнить чеклист по итогам звонка. Контекст использования — сразу после или во время созвона, на десктопе, в браузере. Главная боль — лень и забывчивость: после часового звонка садиться печатать структурированный отчёт никто не хочет.

Время на одну сессию — 5–10 минут. Менеджер задаёт клиенту голосовые вопросы (озвучивает то, что предлагает AI), записывает ответы, подтверждает транскрипцию, получает на выходе готовый Markdown-чеклист, который кладёт в CRM/Notion.

## Product Purpose

Заменить ручное конспектирование созвона на голосовой диалог: AI задаёт 3 вопроса в каждом из 3 раундов (всего 9), пользователь отвечает голосом, локальный Whisper транскрибирует, MiniMax M3 анализирует, второй и третий раунд уточняют пробелы из первого. На выходе — структурированный Markdown с галочками по категориям: общая информация, цели, сроки и бюджет, технические требования, заметки.

Успех = чеклист готов за 5–10 минут без печати, с возможностью отредактировать транскрипт перед отправкой, и MD можно скачать одной кнопкой.

## Brand Personality

**Профессиональный**, **спокойный**, **сфокусированный**. Это рабочий инструмент менеджера, а не игрушка. Тон — ясный, конкретный, без жаргона и эмодзи в копирайте.

Эстетика — editorial-минимализм с тёплой охровой ноткой: один глубокий тёплый brand-цвет несёт «голос» продукта, всё остальное — спокойный нейтрал. Похоже на интерфейс хорошего читалки или редактора заметок, а не на SaaS-дашборд.

## Anti-references

- **AI-gimmick SaaS:** фиолетово-розовые градиенты, sparkle-иконки, «AI Magic ✨» — категорический запрет
- **ChatGPT-чат:** список белых пузырей сообщений — это не разговор, это структурированный опрос
- **Игривый детский UI:** не Duolingo, не Headspace
- **Hero с гигантской метрикой и тремя стат-карточками** — клише SaaS-маркетинга
- **Glassmorphism, гайды gradient-текста, eyebrow-капсы над каждой секцией** — AI-grammar
- **Тёплый cream/sand bg + ochre brand** одновременно — это саттированный AI-аттрактор; тёплый цвет должен быть только в brand, а bg остаётся pure white

## Design Principles

1. **Голос — главный объект экрана.** Кнопка записи, состояние записи и превью транскрипции занимают центр. Всё остальное — поддержка по краям.
2. **Прогресс виден всегда.** Раунд X из 3, вопрос Y из 3 — без догадок, без скрытия в меню. Прогресс-бар или числовой счётчик постоянно на виду.
3. **Подтверждение, а не вера.** Whisper иногда ошибается. После записи показываем транскрипт, разрешаем отредактировать текстом, только потом сабмит. AI — помощник, а не оракул.
4. **Беззвучный успех.** Раунд закрыт — спокойное подтверждение, без салютов, без конфетти. Сложная анимация — это сигнал, что разработчик не уверен в продукте.
5. **Скачиваемый итог.** Конечный артефакт — `.md` файл, не дашборд внутри приложения. Менеджер уходит с файлом и забывает про сервис до следующего созвона.

## Accessibility & Inclusion

- WCAG 2.1 AA минимум. Body 4.5:1, large text 3:1, focus ring всегда виден.
- Полная клавиатурная навигация: запись/стоп — Space, переход между вопросами — Tab.
- `prefers-reduced-motion` — мгновенные переходы вместо анимаций (включая прогресс-бар и кнопки записи).
- `aria-live` для статусов: «запись», «обработка», «готово», «ошибка».
- Поддержка screen reader для всех состояний рекордера (idle / recording / processing / preview / submitted).
- Контент только на русском (см. PRODUCT-решение пользователя). Шрифт должен корректно отображать кириллицу + латиницу для цифр / URL / брендов.

## Language

Русский. Без переключателей языка, без i18n-инфраструктуры. UI копия, промпты LLM, транскрипция Whisper, генерируемый чеклист — всё на русском.
