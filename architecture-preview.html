Как работает голосовой чеклист клиента школы арабского

Менеджер после звонка или переписки с клиентом наговаривает голосом ответы на 10 вопросов про клиента. Локальный Whisper транскрибирует, MiniMax M3 структурирует ответы в чеклист по категориям. На выходе — Markdown-файл для CRM или Telegram-чата команды.

00

Как пользуется этим менеджер школы

Сценарий: менеджер только что закончил разговор с потенциальной ученицей Анной — она написала в Telegram, потом ответила голосовым в WhatsApp, обсудили цели. Менеджер открывает наш инструмент, наговаривает голосом ответы по шаблону из 10 вопросов за 5–7 минут, и получает готовый .md чеклист. Кидает его в командный чат или прикрепляет к карточке клиента в CRM. Никакой регистрации.

Стоп
Идёт запись 00:18
«Клиентку зовут Анна, она из Алматы, часовой пояс GMT плюс пять...»
Что делает Говорит в микрофон 10–40 секунд. Своими словами пересказывает то, что узнал от клиента в Telegram / WhatsApp / по телефону.
Что происходит за кулисами Браузер пишет webm/Opus аудио в память. На сервер пока ничего не уходит. Видна pulse-анимация и таймер.
5

Whisper транскрибирует — короткая пауза

спиннер «Транскрибируем…»
talkarabic.app/session
В1. Имя клиента и где находится?
Транскрибируем ответ…
Аудио → ffmpeg → wav → Whisper
Что делает Ждёт ~3–8 секунд (зависит от длины ответа и мощности backend).
Что происходит за кулисами webm уходит на /api/session/transcribe. Backend конвертирует в wav 16kHz через ffmpeg, прогоняет через локальный Whisper, возвращает текст.

Правит транскрипт (имена клиентов, диалекты, термины)

текстовое поле с черновиком
talkarabic.app/session
Проверьте транскрипцию и при необходимости отредактируйте
Клиентку зовут Анна, она из Алматы, часовой пояс GMT+5.
✓ Подтвердить
↺ Перезаписать
Что делает Читает черновик, поправляет имя клиента или название диалекта если Whisper неправильно расслышал. Подтверждает.
Почему так Whisper иногда косячит на именах собственных и арабских терминах (MSA, фусха, амийя). Дать пользователю быстро поправить дешевле, чем разбираться с кривым чеклистом.

01

Шаблон из 10 вопросов

Фиксированный список — менеджер всегда задаёт одно и то же. LLM не выдумывает новые вопросы, потому что для квалификации клиента в продажах школы важна повторяемость данных от сессии к сессии. Раунды только разбивают шаблон на три удобных «кадра».

Раунд 1 · знакомство и мотивация

Базовая квалификация

  • Имя и где находится (страна, часовой пояс)
  • Откуда узнал о школе (источник лида)
  • Зачем учит арабский (мотивация)
  • Какой диалект — MSA / египетский / левантийский / магрибский / коранический
Раунд 2 · опыт и формат

Подбор программы

  • Текущий уровень владения арабским
  • Опыт онлайн-курсов раньше (что понравилось / не понравилось)
  • Готовность к Zoom-занятиям или асинхронный формат
Раунд 3 · условия и пожелания

Готовность купить

  • Часы в день / неделю на учёбу
  • Бюджет (помесячно или общий)
  • Пожелания: преподаватель м/ж, удобное время, прочее

03

Поток одной сессии

Один цикл — от старта до скачивания. Вопросы фиксированные, LLM используется только для резюме каждого раунда и финального чеклиста. 

sequenceDiagram
    autonumber
    actor U as Менеджер
    participant FE as Frontend
    participant BE as FastAPI
    participant WS as Whisper
    participant LG as LangGraph
    participant LLM as MiniMax M3

    U->>FE: Открывает /session
    FE->>BE: POST /api/session/start
    BE->>BE: questions_for_round(1) — шаблон, без LLM
    BE-->>FE: session_id + 4 вопроса раунда 1

    loop Каждый вопрос (4 в раунде 1, 3 в 2, 3 в 3)
      U->>FE: Запись голоса (MediaRecorder webm)
      FE->>BE: POST /transcribe (audio_file)
      BE->>WS: ffmpeg webm→wav, pipeline()
      WS-->>BE: текст
      BE-->>FE: transcript
      U->>FE: Редактирует и подтверждает
    end

    FE->>BE: POST /{id}/submit (answers)
    BE->>LG: invoke(state)
    LG->>LLM: analyze_round
    LLM-->>LG: round_summary
    alt round < 3
      LG->>LG: questions_for_round(round+1) — шаблон
      LG-->>BE: следующие вопросы + summary
      BE-->>FE: round+1, новые вопросы
    else round == 3
      LG->>LLM: generate_checklist (все 10 ответов + 2 резюме)
      LLM-->>LG: ChecklistItem[]
      LG->>LG: generate_markdown
      LG-->>BE: markdown + items
      BE-->>FE: is_complete=true
    end

    FE->>U: Переход на /results/{id}
    U->>BE: GET /{id}/download
    BE-->>U: checklist-{id}.md
04

LangGraph workflow

После каждого сабмита раунда: analyze_round → условный edge. Раунд < 3 — берём следующие вопросы из шаблона (без LLM). Раунд = 3 — запускаем generate_checklist и форматируем в Markdown.

analyze_round · 1 LLM

Резюме раунда

2–4 предложения о том, что узнали от клиента, и что осталось неясным.

next_questions · 0 LLM

Следующие шаблонные вопросы

Просто берёт вопросы следующего раунда из questions_template.py. Никаких вызовов LLM, ответ мгновенный.

generate_checklist · 1 LLM

Финальный чеклист

Один большой prompt со всеми 10 ответами и 2 резюме. На выходе — структурированный JSON с категориями и статусами.

generate_markdown · 0 LLM

Экспорт MD

Группирует пункты по категориям, добавляет чекбоксы и заметки. Чистый Python, без LLM.

Итого на одного клиента — 3 вызова LLM: 2 резюме (после раундов 1 и 2) + 1 финальный чеклист. ~3–5 тысяч токенов на сессию, ~0.5–1 ₽ через MiniMax M3.

05

Состояния AudioRecorder

Конечный автомат внутри одного компонента. Whisper иногда ошибается, особенно на именах и арабских терминах — между записью и сабмитом пользователь видит черновик и может его поправить руками.

● idle Голубая кнопка с микрофоном. Ждёт нажатия.
■ recording Красная кнопка стоп, моно-таймер, pulse-анимация. MediaRecorder пишет webm/Opus.
↻ processing Spinner. Аудио уехало на /transcribe, ffmpeg → wav, Whisper транскрибирует.
✎ preview Textarea с черновиком. Можно править. Кнопки «Подтвердить» / «Перезаписать».
✓ submitted Зелёная галочка, текст ответа свёрнут в превью. Кнопка «Изменить» возвращает в idle.
06

REST API

Шесть эндпоинтов. Сессии хранятся в памяти процесса; для production менять SessionManager на Redis или SQLite — интерфейс изолирован.

Метод и путь Что делает
POST/api/session/start Создаёт сессию, отдаёт первые 4 вопроса из шаблона (Раунд 1).
POST/api/session/transcribe Принимает один webm-файл, прогоняет через ffmpeg+Whisper, возвращает текст для preview.
POST/api/session/{id}/submit JSON с подтверждёнными транскриптами раунда. Запускает LangGraph. Возвращает либо следующие 3 вопроса, либо is_complete=true + markdown.
GET/api/session/{id}/results Финальный ChecklistItem[] + полный markdown. 409 если сессия не закрыта.
GET/api/session/{id}/download Отдаёт checklist-{id}.md с Content-Disposition: attachment.
GET/health Status + флаг whisper_loaded — для healthcheck.
07

Технологический стек

  • FrontendNext.js 16, React 19, TypeScript, Tailwind 4 (OKLCH tokens), lucide-react, Inter + JetBrains Mono
  • BackendPython 3.11–3.13, FastAPI, Uvicorn, LangGraph 0.2, Pydantic 2
  • LLMminimax/minimax-m3 через OpenRouter (OpenAI SDK 1.59 + base_url)
  • Speech-to-textWhisper Small/Medium локально, через transformers pipeline, ffmpeg для webm→wav 16 kHz
  • Audio (browser)Native MediaRecorder API, webm/Opus
  • ХранениеSQLite (менеджеры + чеклисты), JWT-auth с инвайт-кодами, экспорт в Google Sheets вебхуком
  • Аналитика лидаMiniMax дополнительно отдаёт score 1–10, стадию, возражения, follow-up черновик, дату касания — без лишнего вызова
  • UI токеныOKLCH only. Brand-голубой oklch(0.56 0.15 245), accent-красный oklch(0.56 0.20 25)
  • Дизайн-системаimpeccable skill, PRODUCT.md + DESIGN.md в репозитории
