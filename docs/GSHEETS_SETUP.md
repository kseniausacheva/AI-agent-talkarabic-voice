# Настройка Google Sheets: автоматическая запись чеклистов

Каждый завершённый чеклист автоматически добавляется строкой в вашу
Google-таблицу. Настройка занимает 10 минут и не требует программирования —
просто следуйте шагам по порядку.

## Что получится

В таблице будет появляться строка на каждый завершённый чеклист:

| Дата | Клиент | Менеджер | Резюме | ID сессии |
|---|---|---|---|---|
| 2026-06-10 | Анна | Ксения | Клиентка из Москвы, хочет египетский диалект… | abc123def456 |

## Шаг 1. Создайте таблицу

1. Откройте [sheets.google.com](https://sheets.google.com) и создайте новую таблицу.
2. Назовите её, например, «Чеклисты школы арабского».
3. В первую строку впишите заголовки колонок (по одному в ячейку):
   `Дата` | `Клиент` | `Менеджер` | `Резюме` | `ID сессии`

## Шаг 2. Откройте редактор скриптов

1. В меню таблицы нажмите **Расширения → Apps Script**
   (Extensions → Apps Script, если интерфейс на английском).
2. Откроется новая вкладка с редактором кода.

## Шаг 3. Вставьте код

1. Удалите всё, что есть в редакторе (заготовку `function myFunction() {}`).
2. Скопируйте и вставьте код ниже целиком:

```javascript
// Секретное слово — придумайте своё и впишите между кавычками.
// То же слово нужно указать в настройках сервера (GSHEETS_SECRET).
// Если оставить пустым '' — проверка секрета отключена.
var SECRET = 'moy-sekret-123';

function doPost(e) {
  try {
    var data = JSON.parse(e.postData.contents);

    // Проверка секрета: чужие запросы в таблицу не попадут
    if (SECRET && data.secret !== SECRET) {
      return ContentService
        .createTextOutput(JSON.stringify({ ok: false, error: 'bad secret' }))
        .setMimeType(ContentService.MimeType.JSON);
    }

    var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
    sheet.appendRow([
      data.date,
      data.client_name,
      data.manager,
      data.summary,
      data.session_id
    ]);

    return ContentService
      .createTextOutput(JSON.stringify({ ok: true }))
      .setMimeType(ContentService.MimeType.JSON);
  } catch (err) {
    return ContentService
      .createTextOutput(JSON.stringify({ ok: false, error: String(err) }))
      .setMimeType(ContentService.MimeType.JSON);
  }
}
```

3. В первой строке кода замените `moy-sekret-123` на своё секретное слово
   (любые латинские буквы и цифры, без пробелов). Запомните его — оно
   понадобится на шаге 5.
4. Нажмите значок дискеты (Сохранить) или Ctrl+S.

## Шаг 4. Опубликуйте скрипт как веб-приложение

1. Справа вверху нажмите синюю кнопку **Начать развёртывание → Новое развёртывание**
   (Deploy → New deployment).
2. Нажмите на шестерёнку слева и выберите тип **Веб-приложение** (Web app).
3. Заполните поля:
   - **Описание:** «Чеклисты школы» (любое).
   - **Запуск от имени:** *От моего имени* (Execute as: Me).
   - **У кого есть доступ:** **Все** (Who has access: **Anyone**) — это важно,
     иначе сервер не сможет отправлять данные.
4. Нажмите **Начать развёртывание** (Deploy).
5. Google попросит разрешения: нажмите **Предоставить доступ**, выберите свой
   аккаунт. Если появится предупреждение «Приложение не проверено» — нажмите
   **Дополнительные настройки → Перейти на страницу … (небезопасно)** и
   подтвердите. Это ваш собственный скрипт, он безопасен.
6. После публикации появится **URL веб-приложения** — длинная ссылка вида
   `https://script.google.com/macros/s/AKfycb.../exec`.
   Нажмите «Копировать».

## Шаг 5. Передайте настройки разработчику (или впишите сами)

В файле настроек сервера (`backend/.env`) нужно указать две строки:

```
GSHEETS_WEBHOOK_URL=https://script.google.com/macros/s/AKfycb.../exec
GSHEETS_SECRET=moy-sekret-123
```

- `GSHEETS_WEBHOOK_URL` — ссылка из шага 4.
- `GSHEETS_SECRET` — секретное слово из шага 3 (должно совпадать буква в букву).

После этого перезапустите сервер.

## Шаг 6. Проверьте, что всё работает

Завершите один тестовый чеклист в приложении. В течение нескольких секунд
в таблице должна появиться новая строка. Если строки нет:

- Проверьте, что ссылка в `GSHEETS_WEBHOOK_URL` заканчивается на `/exec`.
- Проверьте, что секрет в `.env` и в коде скрипта совпадает.
- Убедитесь, что при публикации выбран доступ «Все» (Anyone).
- Загляните в логи сервера: строки со словами `GSheets sync`.

## Если нужно изменить код скрипта

После любого изменения кода нужно опубликовать НОВУЮ версию:
**Начать развёртывание → Управление развёртываниями → карандаш →
Версия: Новая версия → Развернуть**. URL при этом не меняется.

## Безопасность

- Ссылка веб-приложения открыта для всех, но без правильного секрета записать
  в таблицу ничего нельзя.
- Скрипт умеет только добавлять строки — читать или удалять данные через него
  невозможно.
- Никому не передавайте секретное слово и ссылку вместе.
